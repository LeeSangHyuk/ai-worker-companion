# ChatGPT Export Batch Review

## Purpose

This document records a batch validation pass over one real ChatGPT Official Export ZIP.

The purpose was to check whether the current import path can scan the full export, identify usable validation samples, export selected conversations into normalized JSONL, and run the existing extractor on a small subset.

This was not a full human usefulness evaluation and not a declaration of complete ChatGPT Export support.

## Exclusion Candidate Interpretation

43 conversations were marked as exclusion candidates, mostly because they were too short.

These should not be used as primary validation samples for Goal Stack / Current Situation / Blocker quality because they provide too little session context.

They may still be useful later for testing edge cases such as:

- one-turn conversations
- empty or near-empty conversations
- assistant-missing conversations
- unusual start state

- running every conversation through an LLM would add cost without materially improving import validation

## What This Validates

Confirmed in this batch:

1. The real export ZIP can be scanned end-to-end.
2. All 124 conversations were parser-importable.
3. Split `conversations-*.json` files work with the current importer.
4. Structural warnings can identify weak validation candidates.
5. 8 representative samples can be exported to normalized JSONL.
6. Existing loader can read all selected JSONL files.
7. Existing extractor can generate Session Representation output for selected real-export samples.

## What This Does Not Validate

Not yet validated:

1. Human usefulness of the generated representations.
2. Exact turn-by-turn fidelity against the ChatGPT UI.
3. Attachment, image, file, or tool artifact handling.
4. Branch reconstruction beyond the selected `current_node` path.
5. Whether warning heuristics are sufficient for all future exports.
6. Whether these selected samples are the best long-term benchmark set.
7. Whether Gemini-generated representations agree with Codex or human review.

## Batch Review Conclusion

The current ChatGPT Official Export import path is strong enough for Phase 0 validation sampling.

The most important finding is that the normalized JSONL boundary held across the full export scan and selected sample exports. The extractor did not require source-specific changes.

However, this is still an import compatibility result, not a product-quality result. The next useful step is human review of the selected representations to determine whether Goal Stack, Current Situation, Blocker, and Evidence actually help someone understand and resume the session quickly.
